# 공모전 제출 범위와 완성도 기준

이 문서의 목적은 범위를 좁히면서도 제출물의 완성도를 높이는 것이다. 이 프로젝트는 장난감 MVP를 만드는 것이 아니라, 핵심 시나리오 하나를 거의 완성형처럼 보이게 만드는 공모전 제출형 v1을 만든다.

## 0. 기본 해석

- 범위는 좁아야 한다.
- 하지만 좁다고 해서 미완성처럼 보이면 안 된다.
- 즉 `넓은 기능 세트`가 아니라 `완성도 높은 핵심 흐름`이 목표다.
- 공모전 심사에서는 구현, 배포, 문서, 리포트, 데모까지 합쳐서 완성도를 본다.

## 1. MVP 대상 문제

- 대상 문제: 제출물이 그럴듯하지만 학생이 실제로 이해했는지 확신하기 어려운 상황
- 대상 맥락: 서술형 과제, 프로젝트 설명문, 짧은 보고서
- 대상 사용자: 교강사 중심

## 2. 첫 버전 핵심 시나리오

### 교사 시나리오

1. 교사가 과제를 등록한다.
2. 교사가 루브릭과 핵심 개념을 입력한다.
3. 학생 제출물을 업로드하거나 붙여넣는다.
4. 시스템이 질문 3개를 생성한다.
5. 학생 답변을 수집한다.
6. 시스템이 분석 결과와 근거를 보여준다.
7. 교사가 최종 판단을 남긴다.

이 흐름은 데모용이 아니라 실제 제출 제품 수준으로 매끈하게 동작해야 한다.

### 학생 시나리오

1. 학생이 제출물에 대해 생성된 질문을 확인한다.
2. 학생이 텍스트 또는 음성으로 짧게 답변한다.
3. 학생은 결과 전체가 아니라 필요한 범위의 피드백을 본다.

### 운영자 시나리오

1. 운영자는 여러 학생 결과를 집계 형태로 본다.
2. 어떤 개념이 반복적으로 무너지는지 확인한다.

## 3. P0 필수 기능

- 과제/검증 세션 생성
- 루브릭 입력
- 제출물 입력
- 질문 3개 자동 생성
- 학생 답변 입력
- AI 분석 결과 생성
- 교사용 결과 뷰
- 교사 최종 판단 저장
- 결과 내보내기 JSON/CSV

## 4. 공모전 제출형 v1 완성도 조건

아래는 있으면 좋은 수준이 아니라 사실상 필수다.

- 배포 URL에서 바로 동작
- 샘플 데이터로 즉시 데모 가능
- 교사 기준 핵심 시나리오 끊김 없음
- 질문 3개가 각각 역할이 분명함
- 결과가 단순 점수 아닌 근거 중심
- 에러 시 빈 화면이 아니라 재시도 또는 안내 제공
- 저장소 문서와 실제 화면 흐름 일치
- AI 리포트와 실제 구현 로직 일치
- 발표 시 보여줄 스크린샷과 export 예시 확보

## 5. P1 있으면 좋은 기능

- 음성 입력 후 STT 변환
- 질문 재생성 1회
- 반 단위 오개념 간단 집계
- 분석 실패 재시도
- 샘플 데이터 불러오기

## 6. P2 이후로 미루는 기능

- LMS 완전 연동
- LTI 1.3 통합
- 고급 권한 체계
- 실시간 협업
- 영상 면접 스타일 UI
- 전 과목 범용화
- 자동 점수 산출
- AI 작성 여부 탐지 점수

## 7. 첫 버전 입력 범위

권장 입력 형식:

- 과제 제목
- 과제 설명
- 제출물 본문 텍스트 또는 텍스트 추출본
- 핵심 개념 목록
- 금지 포인트 또는 자주 틀리는 포인트

권장 답변 형식:

- 텍스트 우선
- 음성은 선택
- 답변 길이는 90초 내외 또는 이에 상응하는 짧은 서술

## 8. 첫 버전 출력 범위

교사용 필수 출력:

- 이해 신뢰도
- 결과 분류
- 판단 근거 요약
- 빠진 개념
- 충돌 문장
- 재설명 포인트
- 교사 최종 판단 입력

운영자용 필수 출력:

- 과제별 결과 분포
- 반복 오개념 태그 빈도

## 9. 데모용 성공 지표

- 질문 3개가 서로 역할이 다르다.
- 질문이 제출물 기반으로 보인다.
- 학생 답변에 따라 결과가 달라진다.
- 결과에 근거가 포함된다.
- 교사가 마지막 판단을 직접 누를 수 있다.
- JSON/CSV export가 실제로 된다.
- 화면과 문서가 공모전 제출물 수준으로 정리돼 있다.

## 10. 의도적으로 하지 않을 것

- 멋진 로그인 시스템
- 많은 화면 수
- 범용 관리자 기능
- 고급 차트
- 추천/학습 보조 기능
- 대규모 데이터 파이프라인
- 실사용 조직 배포 수준의 권한 체계

## 11. 샘플 데모 데이터 기준

데모 데이터는 아래 성질을 가진 것이 좋다.

- 개념 이해 여부가 드러나는 과제
- 전이 질문이 자연스럽게 가능한 과제
- 반례 질문이 가능한 과제
- 제출물은 괜찮아 보이지만 답변에서 오개념이 드러날 수 있는 사례

## 12. 범위 통제 질문

기능을 추가하기 전 아래 질문을 통과해야 한다.

- 이것이 질문 생성 또는 분석 품질을 높이는가
- 이것이 교사의 최종 판단을 더 쉽게 만드는가
- 이것이 데모 메시지를 더 선명하게 만드는가
- 이것이 마감 전 안정적으로 구현 가능한가
- 이것이 제출물 완성도를 실제로 높이는가

하나라도 아니면 후순위로 미룬다.
