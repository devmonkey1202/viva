# v1 기능 명세

이 문서는 공모전 제출형 v1에서 실제로 구현해야 할 기능을 구체적으로 적는다.

## 1. 핵심 사용자

- 교강사
- 학생
- 교육 운영자

우선순위는 교강사 기준 흐름이 가장 높다.

## 2. 핵심 사용자 흐름

### 교사 흐름

1. 과제 생성
2. 루브릭 입력
3. 제출물 확인
4. 질문 생성
5. 학생 답변 확인
6. 분석 결과 확인
7. 최종 판단 저장
8. 결과 export

### 학생 흐름

1. 질문 확인
2. 짧은 답변 제출
3. 제출 완료 확인

### 운영자 흐름

1. 누적 결과 확인
2. 오개념 분포 확인

## 3. 필수 화면

- 홈 또는 대시보드
- 과제/검증 세션 생성 화면
- 루브릭 입력 화면
- 제출물 상세 화면
- 질문 생성 결과 화면
- 학생 답변 입력 화면
- 분석 결과 화면
- 운영자 요약 화면

## 4. 필수 기능 상세

### 과제 생성

- 과제 제목 입력
- 과제 설명 입력
- 과제 유형 선택

### 루브릭 입력

- 학습 목표
- 핵심 개념 목록
- 금지 포인트 또는 자주 틀리는 포인트

### 제출물 입력

- 텍스트 붙여넣기
- 샘플 데이터 불러오기

### 질문 생성

- 왜형 질문 1개
- 전이형 질문 1개
- 반례형 질문 1개
- 질문 생성 근거 또는 생성 메모

### 학생 답변

- 텍스트 입력
- 답변 길이 안내
- 제출 완료 상태

### 분석 결과

- 결과 분류
- 이해 신뢰도
- 판단 근거 요약
- 빠진 개념
- 충돌 문장
- 재설명 포인트

### 교사 최종 판단

- 승인 또는 보류 성격의 판단
- 메모 입력
- 최종 저장

### export

- JSON export
- CSV export

## 5. 필수 API 또는 서비스 단위

- create-assignment
- save-rubric
- save-submission
- generate-question-set
- save-answer-session
- run-analysis
- save-teacher-decision
- export-results

## 6. 검증 규칙

- 질문은 3축을 모두 포함해야 한다.
- 분석 결과는 스키마를 통과해야 한다.
- 교사 판단 없이 최종 점수 확정은 없다.
- 불확실 사례를 허용해야 한다.

## 7. 비기능 요구

- 입력이 길어도 흐름이 깨지지 않아야 한다.
- 실패 시 재시도 또는 안내가 있어야 한다.
- 샘플 데이터로 데모가 가능해야 한다.
- 공개 저장소 기준으로 민감정보가 없어야 한다.

## 8. 데모용 필수 아티팩트

- 샘플 루브릭
- 샘플 제출물
- 샘플 답변
- 분석 결과 예시
- export 파일 예시
- 화면 스크린샷

## 9. 구현 순서

1. 스키마
2. 프롬프트
3. 샘플 데이터
4. 서비스 함수
5. 화면
6. export
