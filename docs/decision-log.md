# 의사결정 로그

이 문서는 중요한 제품/기술 결정을 짧게 남기는 로그입니다.

## 2026-04-10

- 제품 정체성을 `평가 검증 레이어`로 고정했다.
  - 이유: 단순 LMS나 AI 튜터보다 원본 아이디어와 공모전 방향에 더 직접적으로 맞는다.

- 최종 판단은 교사 승인 구조를 유지하기로 했다.
  - 이유: 공정성과 설명 책임을 AI가 아닌 교사에게 두기 위해서다.

- 제출물 수준은 `장난감 MVP`가 아니라 `공모전 제출형 v1`로 정의했다.
  - 이유: 이 대회는 구현, 배포, 문서, 리포트까지 함께 본다.

## 2026-04-11

- 제품명은 `VIVA`로 확정했다.
  - 이유: 외부 제품명으로 더 짧고 강하며 발표/심사에서 설명이 쉽다.

- AI 전략은 `기반 모델 직접 학습`보다 `VIVA 전용 검증 시스템 직접 구현`으로 잡았다.
  - 이유: 현재 단계에서 더 중요한 것은 배포 가능한 구조, 설명 가능한 결과, 운영 가능한 품질 관리다.

- 저장 구조는 `로컬 JSON + Neon/Postgres` 하이브리드로 잡았다.
  - 이유: 로컬 개발 속도와 배포 안정성을 동시에 확보하기 위해서다.

- 배포 경로는 `Vercel + Neon(Postgres)`를 기본으로 잡았다.
  - 이유: 무료 기준에서 가장 현실적인 조합이기 때문이다.

## 2026-04-13

- OpenAI Responses API는 `json_object` 기반 구조화 응답으로 고정했다.
  - 이유: 자유 텍스트 파싱보다 재현성과 실패 통제가 높다.

- STT는 브라우저 Web Speech API + 텍스트 fallback으로 구현했다.
  - 이유: 무료 시연 가능성과 원문 문서 정합성을 동시에 맞추기 위해서다.

- 자동 검증은 `lint + build + HTTP smoke test + CI` 조합으로 강화했다.
  - 이유: 로컬에서만 되는 느낌이 아니라 재실행 가능한 검증 흐름이 필요했기 때문이다.

- 교사 화면에 학생 답변과 전사 메타 검토 영역을 추가했다.
  - 이유: 최종 판단 전 실제 학생 답변을 직접 확인할 수 있어야 하기 때문이다.
