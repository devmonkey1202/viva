## 문서 운영 원칙

- 문서와 코드가 충돌하면 원문 규정과 구현 상태를 다시 대조한다.
- 같은 내용은 한 문서에만 적고, 다른 곳은 링크만 둔다.
- 디자인, 기능, 제출은 각각 기준 문서 하나에 모은다.
- 기록 문서는 작업 흐름이 바뀔 때만 갱신한다.
