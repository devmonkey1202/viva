# 저장소 구조 기준

이 문서는 구현 시작 전 저장소를 어떤 경계로 나눌지 정의한다. 특정 프레임워크를 강제하지는 않지만, 어떤 기술 스택을 쓰더라도 이 책임 분리는 유지해야 한다.

## 1. 권장 최상위 구조

```text
/
  README.md
  PROJECT_RULES.md
  WORKBOARD.md
  docs/
  reports/
  prompts/
  schemas/
  samples/
  artifacts/
  apps/
    web/
    api/
  packages/
    domain/
    ai/
    shared/
    ui/
  scripts/
  tests/
```

## 2. 각 폴더의 책임

### `docs/`

- 기준 문서
- 의사결정 로그
- 대회 제출 운영 문서
- AI 리포트 작성 가이드

### `reports/`

- AI 리포트 초안
- 진행 요약
- 제출 직전 문서 초안

### `prompts/`

- 질문 생성 프롬프트
- 답변 분석 프롬프트
- 요약/클러스터링 프롬프트
- 프롬프트 버전 관리 파일

### `schemas/`

- AI 출력 JSON schema
- API 요청/응답 계약
- 내보내기 포맷 정의

### `samples/`

- 익명화된 루브릭 샘플
- 제출물 샘플
- 학생 답변 샘플
- 기대 출력 샘플

### `artifacts/`

- 데모 스크린샷
- 발표용 캡처
- 예시 리포트
- 제출 직전 생성한 PDF/CSV/JSON 예시

### `apps/web/`

- 학생/교사용 화면
- 폼, 업로드, 결과 뷰
- 접근 제어가 필요한 경우의 웹 레이어

### `apps/api/`

- 세션 생성 API
- 질문 생성 API
- 답변 분석 API
- 내보내기 API
- 저장/조회 API

### `packages/domain/`

- 도메인 엔터티
- 판정 규칙
- 상태 전이
- 도메인 서비스 인터페이스

### `packages/ai/`

- 프롬프트 로더
- 모델 호출기
- 출력 파서
- 재시도 정책
- 근거 정리 유틸리티

### `packages/shared/`

- 공통 타입
- 유틸리티
- 에러 객체
- 공통 상수

### `packages/ui/`

- 재사용 UI 컴포넌트
- 디자인 토큰
- 상태 배지, 결과 카드 등 공통 조각

### `scripts/`

- 샘플 데이터 주입
- PDF/텍스트 변환
- 제출 산출물 생성 보조 스크립트

### `tests/`

- 유닛 테스트
- 계약 테스트
- 대표 시나리오 E2E 테스트

## 3. 코드가 적을 때의 축소형 구조

초기 구현에서 모노레포가 부담되면 단일 앱 구조로 축소할 수 있다. 다만 폴더 이름만 줄일 뿐 책임 분리는 유지한다.

```text
/
  src/
    app/
    domain/
    ai/
    shared/
    ui/
```

이 경우에도 아래는 반드시 분리한다.

- 화면 레이어
- 도메인 규칙
- AI 호출과 파싱
- 외부 I/O

## 4. 파일 네이밍 규칙

- 문서 파일은 소문자 케밥 케이스를 사용한다.
- 프롬프트는 `purpose.v1.md` 형식으로 버전 관리한다.
- 스키마는 `entity.schema.json` 형식으로 저장한다.
- 샘플은 `assignment-a.sample.json`처럼 목적이 드러나게 이름 짓는다.
- 임시 파일 이름에 `final`, `real`, `new2` 같은 표현을 남발하지 않는다.

## 5. 모듈 경계 규칙

- `web`는 `ai provider`를 직접 모르면 좋다.
- `domain`은 UI 프레임워크를 몰라야 한다.
- `domain`은 특정 모델명과 프롬프트 파일 경로를 몰라야 한다.
- `ai`는 도메인 규칙을 덮어쓰지 않는다. 도메인 판정 구조를 따르는 보조자다.
- `shared`는 저수준 공통 코드만 가진다. 잡탕 폴더가 되면 안 된다.

## 6. 추천 산출물 저장 규칙

- 프롬프트는 `prompts/`에 저장한다.
- AI 출력 스키마는 `schemas/`에 저장한다.
- 샘플 입력/출력은 `samples/`에 저장한다.
- 심사 증빙 이미지와 자료는 `artifacts/`에 저장한다.
- 대회용 PDF와 제출 파일은 별도 제출 직전 폴더로 복사하고, 공개 저장소에는 개인정보 없는 버전만 남긴다.

## 7. 폴더 구조 의사결정 원칙

다음 질문에 답하지 못하면 새 폴더를 만들지 않는다.

- 이 폴더는 다른 폴더와 책임이 다른가
- 이 폴더는 두 달 뒤에도 유효한가
- 이 폴더가 없으면 경계가 실제로 무너지는가

그렇지 않다면 기존 경계 안에 넣는다.
