# VIVA 기능 감사 및 서비스 운영 계획

이 문서는 현재 VIVA 기능을 `문서 기준`, `코드 기준`, `자동 검증 기준`, `실사용 기준`으로 교차 검증한 결과와, 실서비스 수준으로 가기 위해 반드시 필요한 편의 기능과 운영 기능을 정리한다.

## 1. 검증 기준

기능은 아래 네 가지 기준으로 본다.

- 문서 기준: 제품 정의와 기능 명세에 맞는가
- 코드 기준: 실제 라우트, 컴포넌트, 저장 구조가 구현되어 있는가
- 자동 검증 기준: unit test, smoke test, build, CI로 반복 검증 가능한가
- 실사용 기준: 사용자가 흐름을 끊김 없이 수행할 수 있는가

## 2. 현재 교차 검증 결과

### A. 제품 핵심 흐름

#### 1. 교사 입력 -> 질문 생성

- 상태: 구현 및 검증 완료
- 문서 기준: 충족
- 코드 기준: `/teacher`, `/api/questions`
- 자동 검증 기준: `verify`, `smoke:http`, `smoke:url`
- 실사용 기준: 가능
- 메모: production에서는 현재 OpenAI 크레딧 문제로 `mock-engine` fallback이 발생할 수 있다

#### 2. 학생 링크 진입 -> 질문 확인

- 상태: 구현 및 검증 완료
- 문서 기준: 충족
- 코드 기준: `/student/[verificationId]`, `/api/verifications/[verificationId]`
- 자동 검증 기준: `smoke:http`, `smoke:url`
- 실사용 기준: 가능

#### 3. 학생 텍스트 답변 제출

- 상태: 구현 및 검증 완료
- 문서 기준: 충족
- 코드 기준: student flow, `/api/analyze`
- 자동 검증 기준: `student-answer-flow.test.ts`, `smoke:http`, `smoke:url`
- 실사용 기준: 가능

#### 4. 학생 음성 답변 + STT fallback

- 상태: 구현 완료, 브라우저 조건부
- 문서 기준: 충족
- 코드 기준: `Web Speech API + text fallback`
- 자동 검증 기준: 전사 정규화 로직 unit test 존재
- 실사용 기준: Chrome/Edge 계열에서 강함, 미지원 브라우저에서는 텍스트 fallback
- 메모: 완전한 서버형 STT가 아니라 브라우저 의존 경로다

#### 5. 분석 결과 생성

- 상태: 구현 및 검증 완료
- 문서 기준: 충족
- 코드 기준: `/api/analyze`, analysis schema
- 자동 검증 기준: `smoke:http`, `smoke:url`
- 실사용 기준: 가능
- 메모: 실AI 경로는 크레딧 이슈 해결 후 재검증 필요

#### 6. 교사 최종 판단 저장

- 상태: 구현 및 검증 완료
- 문서 기준: 충족
- 코드 기준: `/api/teacher-decisions`
- 자동 검증 기준: `smoke:http`, `smoke:url`
- 실사용 기준: 가능

#### 7. 운영자 요약 보기

- 상태: 구현 및 검증 완료
- 문서 기준: 충족
- 코드 기준: `/operator`, `/api/summary`
- 자동 검증 기준: `smoke:http`, `smoke:url`
- 실사용 기준: 가능

#### 8. JSON/CSV export

- 상태: 구현 및 검증 완료
- 문서 기준: 충족
- 코드 기준: `/api/export`
- 자동 검증 기준: `smoke:http`, `smoke:url`
- 실사용 기준: 가능

#### 9. 최근 세션 검색 / 재오픈

- 상태: 구현 및 검증 완료
- 문서 기준: 충족
- 코드 기준: `/api/verifications`, session browser
- 자동 검증 기준: `verification-list.test.ts`, `smoke:http`, `smoke:url`
- 실사용 기준: 가능

### B. 제품 운영에 필요한 기본 기능

#### 10. health check / 런타임 상태 확인

- 상태: 구현 및 검증 완료
- 코드 기준: `/api/health`
- 자동 검증 기준: `smoke:http`, `smoke:url`
- 실사용 기준: 운영 점검 가능

#### 11. AI fallback 가시성

- 상태: 부분 완료
- 코드 기준: fallback reason 기록
- 자동 검증 기준: 간접 확인 가능
- 실사용 기준: teacher가 모델 버전과 caution note를 통해 파악 가능
- 메모: production 운영 화면에서 더 직접적으로 보여줄 필요가 있다

#### 12. 에러 응답 구조화

- 상태: 구현 완료
- 코드 기준: `createApiErrorResponse`
- 자동 검증 기준: 스모크에서 간접 검증
- 실사용 기준: 기본 수준 충족

## 3. 아직 없는 기능

아래는 `문서상 필요하거나`, `실서비스 기준으로 사실상 필수인데`, 아직 라우트나 UI로 분리되지 않은 것들이다.

### 반드시 추가해야 하는 기능

#### 1. 로그인 페이지 `/login`

- 현재 상태: 없음
- 이유: 교사/운영자 인증 surface를 실제 제품 수준으로 만들려면 진입 페이지가 있어야 한다

#### 2. 설정 페이지 `/settings`

- 현재 상태: 없음
- 이유: 계정, AI, 저장, export, 개인정보 정책을 숨기지 말고 한 곳에 모아야 한다

#### 3. 교사 세션 상세 페이지 `/teacher/verifications/[verificationId]`

- 현재 상태: 없음
- 이유: 현재 `/teacher` 한 화면에 너무 많은 역할이 몰려 있다
- 가치: deep-link, review, export, later collaboration에 유리하다

#### 4. 세션 만료 / 권한 없음 / 접근 불가 상태 페이지

- 현재 상태: 없음
- 이유: student 링크 기반 구조에서는 만료와 접근 제어 안내가 반드시 필요하다

#### 5. 실질적 인증/권한 모델

- 현재 상태: 문서만 있고 구현 없음
- 이유: 지금은 누구나 `/teacher`, `/operator` 접근이 가능하다
- 리스크: 실서비스 기준으로는 치명적

### 강하게 권장되는 기능

#### 6. 초안 자동 저장

- 현재 상태: 없음
- 이유: 교사 입력이 길다. 새로고침이나 이탈 시 손실되면 사용성이 크게 떨어진다

#### 7. 이탈 방지 경고

- 현재 상태: 없음
- 이유: 입력이 길고 분석 단계가 있으므로 unsaved changes 경고가 필요하다

#### 8. 세션 상태 필터

- 현재 상태: 없음
- 예시 상태:
  - 초안
  - 질문 생성됨
  - 학생 답변 대기
  - 분석 완료
  - 교사 판단 완료
- 이유: 세션이 늘면 검색만으로는 운영이 어렵다

#### 9. 재분석 버튼

- 현재 상태: 없음
- 이유: 학생 답변 수정이나 모델 전략 변경 시 재실행 경로가 필요하다

#### 10. 링크 재발급 / 링크 비활성화

- 현재 상태: 없음
- 이유: student 링크 기반 구조에서는 공유와 회수가 둘 다 있어야 한다

#### 11. 세션 삭제 또는 보관

- 현재 상태: 없음
- 이유: 운영상 불필요 세션 정리 수단이 필요하다

#### 12. export 범위 선택

- 현재 상태: 전체 export만 존재
- 이유: 단일 세션 export, 현재 검색 결과 export, 전체 export를 나눌 필요가 있다

## 4. 편의 기능 카탈로그

### 교사용 편의 기능

#### 반드시 필요한 편의 기능

- 최근 세션 검색
- 최근 세션 재오픈
- student 링크 복사
- 질문 생성 전 입력 검증
- 분석 실행 중 상태 표시
- 최종 판단 저장 후 즉시 피드백
- 학생 전사 메타 확인
- 모델 fallback 여부 확인

#### 추가하면 강한 편의 기능

- 루브릭 템플릿 저장 / 불러오기
- 이전 과제 복제
- 샘플 데이터 빠른 주입
- 세션 정렬
- 세션 상태 필터
- 결과 요약 복사
- 분석 근거 펼치기 / 접기
- export 직전 미리보기

### 학생용 편의 기능

#### 반드시 필요한 편의 기능

- 질문 3개를 한 화면에서 명확히 보기
- 모바일에서 큰 입력 영역
- STT 지원 여부 안내
- 전사 실패 시 텍스트 fallback
- 제출 완료 상태 명확히 보여주기
- 이미 제출됨 상태 안내

#### 추가하면 강한 편의 기능

- 진행률 표시
- 답변 길이 가이드
- 음성 입력 중 상태 애니메이션
- 답변 임시 저장
- 재제출 가능 여부 안내

### 운영자용 편의 기능

#### 반드시 필요한 편의 기능

- 최근 세션 목록
- 분류 분포
- 교사 판단 분포
- 상위 오개념
- 상위 누락 개념

#### 추가하면 강한 편의 기능

- 기간 필터
- 과제명 검색
- 분류별 drill-down
- teacher decision별 drill-down
- CSV export of summary

## 5. 기본 운영 서비스 관점에서 필요한 항목

기능이 동작하는 것과 서비스가 운영되는 것은 다르다. 아래는 운영 서비스로서 필요한 항목이다.

### 필수 운영 기능

- health endpoint
- 구조화된 error response
- 배포 환경 변수 관리
- DB 연결 상태 확인
- AI fallback 감지
- 로그 확인 경로
- CI
- build/lint/test/smoke 재현 가능성

### 필수 운영 페이지 또는 정책

- 로그인
- 로그아웃
- 설정
- 권한 없음 안내
- 세션 만료 안내
- 개인정보 보관 정책
- 브라우저/STT 지원 정책

### 강하게 권장되는 운영 기능

- 감사 로그
- 요청 ID 노출
- 운영자용 문제 세션 모니터링
- 에러 빈도 요약
- 사용량/실패율 요약

## 6. 현재 완성도 판단

### 잘 닫힌 영역

- 핵심 제품 흐름
- mock 기준 end-to-end 검증
- 배포
- DB 연결
- STT + fallback
- export
- 최근 세션 검색

### 아직 100이 아닌 영역

- 인증과 권한
- 설정
- 세션 상세 페이지 분리
- 라이브 AI 운영 검증
- 운영성 가시화
- 고급 편의 기능

## 7. 폐기 리스크가 되는 항목

사용자 기준에서 아래 중 하나라도 실제 데모에서 크게 깨지면 프로젝트 신뢰도가 크게 무너진다.

- 교사 입력 -> 질문 생성이 안 된다
- student 링크 진입이 안 된다
- 학생 답변 제출이 안 된다
- 분석 결과 저장이 안 된다
- 교사 판단 저장이 안 된다
- operator 요약이 안 된다
- export가 안 된다
- 에러가 났는데 사용자가 이유를 알 수 없다
- fallback이 발생했는데 운영자가 모른다

## 8. 지금 당장 우선순위

### 구현 우선

1. `/login`
2. `/settings`
3. `/teacher/verifications/[verificationId]`
4. 권한 없음 / 만료 상태 페이지

### 기능 우선

1. 입력 초안 저장
2. 이탈 방지
3. 세션 상태 필터
4. 재분석 버튼
5. 링크 비활성화 또는 재발급

### 운영 우선

1. fallback 상태를 더 직접적으로 노출
2. 요청 실패 로그를 운영자가 확인하기 쉽게 정리
3. 라이브 AI 경로 재검증 준비
