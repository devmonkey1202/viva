# 구현 로드맵

이 문서는 공모전 제출형 v1을 실제로 만들기 위한 단계별 실행 로드맵이다.

## 목표

- 넓은 기능군이 아니라 핵심 검증 시나리오 하나를 거의 완성형 수준으로 구현한다.
- 구현, 배포, 문서, 리포트를 동시에 맞춘다.

## Phase 0. 기준 고정

상태:

- 완료

완료 항목:

- 문제 정의
- 대회 기준 정리
- 아키텍처
- 도메인 모델
- 클린 코드 규칙
- 제출 운영 규칙

산출물:

- 기준 문서 세트

## Phase 1. 데이터 계약과 AI 계약

목표:

- 나중에 UI와 API가 흔들리지 않게 입력/출력 구조를 먼저 고정한다.

작업:

- 질문 세트 스키마 작성
- 분석 리포트 스키마 작성
- 교사 판단 스키마 작성
- export 행 스키마 작성
- 질문 생성 프롬프트 v1 작성
- 답변 분석 프롬프트 v1 작성
- 샘플 입력/출력 세트 작성

완료 기준:

- 스키마가 문서와 일치한다.
- 프롬프트가 스키마 기반으로 동작한다.
- 샘플 데이터로 결과 예시를 만들 수 있다.

## Phase 2. 핵심 Happy Path 구현

목표:

- 교사 기준 메인 시나리오를 처음부터 끝까지 연결한다.

작업:

- 과제/루브릭 입력 화면
- 제출물 입력 또는 로딩
- 질문 생성 실행
- 학생 답변 입력
- 분석 실행
- 결과 뷰
- 교사 최종 판단 저장

완료 기준:

- 한 세션이 처음부터 끝까지 끊기지 않는다.
- 빈 화면이나 막힌 흐름이 없다.
- 저장과 조회가 된다.

## Phase 3. 결과 품질과 운영성 보강

목표:

- 데모가 아니라 제출 가능한 제품처럼 보이게 다듬는다.

작업:

- 결과 근거 시각화 정리
- 빠진 개념/충돌 문장 표시
- 실패 상태와 재시도 처리
- export 구현
- 운영자 요약 뷰 최소 구현

완료 기준:

- 결과가 점수 중심이 아니라 근거 중심이다.
- 에러 시 재시도 경로가 있다.
- export가 실제 제출 데모에 쓸 수 있다.

## Phase 4. 문서/리포트/배포 정합화

목표:

- 코드와 문서와 리포트가 하나의 이야기처럼 맞물리게 만든다.

작업:

- README 정리
- AI 리포트 본문 정리
- 스크린샷 캡처
- 샘플 export 저장
- 배포 URL 안정화
- 제출 체크리스트 점검

완료 기준:

- 저장소 설명과 실제 제품이 일치한다.
- AI 리포트 설명과 구현이 일치한다.
- 제출물 5종이 준비된다.

## 우선순위 원칙

- 먼저 스키마
- 다음 프롬프트
- 다음 샘플 데이터
- 다음 핵심 화면/API
- 마지막에 꾸미기

## 절대 미루면 안 되는 것

- 배포 검증
- export 확인
- AI 리포트 초안 작성 시작
- 비밀키/민감정보 점검

## 절대 끼워 넣지 말 것

- 전 과목 지원
- 자동 채점
- 과한 관리자 기능
- 영상 중심 흐름
- AI 탐지 점수
