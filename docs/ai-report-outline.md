# AI 리포트 작성 가이드

공식 AI 리포트 양식에 맞춰 어떤 내용을 써야 하는지 미리 구조화한 문서다. 최종 제출 시 이 문서를 바탕으로 PDF를 작성한다.

## 1. 기본 정보

양식에 들어가는 기본 필드:

- 팀명 또는 이름
- 대표자 연락처
- 프로젝트명

주의:

- 저장소 문서에는 실제 개인 연락처를 반복 기재하지 않는다.
- 제출용 DOCX/PDF에서만 실제 정보를 입력한다.

## 2. 1번 섹션: 기획

### 2-1. 설정한 사용자는 누구이며, 해결하려는 구체적인 문제점/불편함은 무엇인가요?

권장 작성 구조:

- 1차 사용자: 교강사
- 2차 사용자: 학생/수강생
- 3차 사용자: 교육 운영자
- 문제의 본질: 결과물만으로는 이해를 확신하기 어려움
- AI 보편화로 인해 결과물과 실제 이해 수준이 더 쉽게 분리됨
- 교사는 개별 구술을 모두 할 시간이 부족함
- 운영자는 학습 품질을 설명할 데이터가 부족함

핵심 문장 후보:

- VIVA는 학습 자체가 아니라 학습 검증의 실패를 해결한다.

### 2-2. 문제를 해결하기 위한 솔루션의 핵심 기능은 무엇인가요?

권장 작성 구조:

- 제출물 업로드
- 루브릭 및 핵심 개념 입력
- 학생별 질문 3개 자동 생성
- 학생의 짧은 답변 수집
- 제출물-루브릭-답변 삼자 비교 분석
- 이해 신뢰도, 빠진 개념, 오개념, 충돌 문장 제공
- 교사 최종 판단
- 운영자용 집계

핵심 문장 후보:

- VIVA는 제출 이후 검증 레이어로 작동한다.

### 2-3. 이 솔루션이 도입되었을 때 기대되는 개선점이 무엇인가요?

권장 작성 포인트:

- 교사는 장시간 전수 구술 없이도 더 정교한 검증 가능
- 학생은 AI 사용 여부 논쟁보다 실제 이해를 증명할 수 있음
- 운영자는 과정 품질을 이해 검증 데이터로 파악 가능
- 결과물 품질과 실제 이해를 분리해서 볼 수 있음

## 3. 2번 섹션: AI 활용 전략

### 3-1. 어떤 AI 도구와 모델을 사용했고, 선택 이유는 무엇인가요?

작성 원칙:

- 실제 사용한 도구만 적는다.
- 도구별 역할이 겹치지 않게 쓴다.
- 선택 이유는 품질, 속도, 비용, 작업 적합성 관점에서 설명한다.

권장 구조:

- 기획/문서 정리용 도구
- 코드 구현 보조 도구
- 런타임 질문 생성/분석용 모델

### 3-2. 각 AI 도구별 활용 전략

양식이 요구하는 포인트:

- 도구/모델별 페르소나
- 핵심 전략
- 제약 사항
- 에이전트 구성 방식
- 데이터 흐름

권장 서술 예시 구조:

- 문서/기획 에이전트
  문제정의, 범위 축소, 리포트 초안 정리
- 구현 에이전트
  폴더 구조, 코드 작성, 테스트 보조
- 런타임 분석 모델
  질문 생성, 의미 비교, 근거 구조화

### 3-3. 토큰 낭비 최소화와 유지보수성/재현성 전략

반드시 들어가야 할 포인트:

- 경량 모델과 고성능 모델의 역할 분리
- 프롬프트 버전 관리
- 출력 스키마 고정
- 샘플 데이터 기반 회귀 확인
- 모델 버전/실행 로그 저장
- 불확실 사례만 정밀 분석

## 4. 리포트에서 반드시 살아야 하는 메시지

- 이 프로젝트는 단순 LMS가 아니다.
- 이 프로젝트는 AI 탐지기가 아니다.
- 이 프로젝트는 AI 시대 평가 패러다임 재설계에 가깝다.
- AI가 필요한 이유는 개인별 질문 생성과 의미 비교 때문이다.
- 교사가 최종 판단을 가지므로 안전장치가 있다.
