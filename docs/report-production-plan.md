# 리포트 생산 계획

이 문서는 AI 리포트와 기타 제출용 문서를 어떻게 병행 생산할지 정리한다.

## 1. 만들어야 하는 문서 산출물

- README
- AI 리포트 초안
- 진행 요약
- 아키텍처 문서
- AI 전략 문서
- 제출 체크리스트
- 샘플 결과 자료

## 2. AI 리포트 작성 방식

- 구현이 끝난 뒤 한 번에 쓰지 않는다.
- 구현과 같이 계속 갱신한다.
- 공식 양식 질문에 대응하는 문장을 미리 축적한다.

## 3. 심사 기준별 증빙 자료

### 기술적 완성도

- 실제 화면 캡처
- 흐름 설명
- 배포 URL
- export 예시

### AI 활용 능력 및 효율성

- 프롬프트 구조
- 스키마 기반 출력
- 모델 역할 분리
- 비용 절감 전략

### 기획력 및 실무 접합성

- 교사 pain
- 학생 pain
- 운영자 pain
- 실제 사용 시나리오

### 창의성

- LMS와의 차이
- AI 튜터와의 차이
- AI 탐지기와의 차이

## 4. 리포트 생산 순서

1. 문제 정의 문장 확정
2. 기능 설명 문장 확정
3. AI 전략 문장 확정
4. 구현 화면과 결과 캡처
5. 리포트 본문 정리
6. PDF 변환 전 최종 교정

## 5. 리포트에 반영해야 하는 실제 구현 항목

- 질문 3개 구조
- 학생 답변 수집 방식
- 분석 기준
- 교사 최종 판단 구조
- export 기능
- 배포 방식

## 6. 리포트에 과장 없이 쓰는 방식

- 이미 구현된 것: 현재형
- 제출 전 확정된 것: 현재형 또는 완료형
- 향후 확장: 미래형

## 7. 계속 모아둘 증빙

- 주요 화면 스크린샷
- 샘플 입력/출력
- export 파일
- 프롬프트 버전 정보
- 모델 사용 메모

## 8. 제출 직전 확인

- README와 AI 리포트 메시지 일치
- 제품 설명과 화면 흐름 일치
- AI 사용 설명과 실제 구현 일치
- 과장 표현 제거
