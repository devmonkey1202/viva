# AI 활용 전략

이 문서는 공모전 심사 기준 중 `AI 활용 능력 및 효율성`에 직접 대응한다. 단순히 모델을 붙였다는 설명이 아니라, 어떤 역할을 어디에 왜 배치하는지까지 정리한다.

## 1. AI가 맡는 역할

- 제출물과 루브릭을 바탕으로 학생별 질문을 생성한다.
- 학생 답변을 제출물/루브릭과 의미 수준에서 비교한다.
- 누락 개념, 논리 충돌, 전이 실패, 오개념 후보를 구조화한다.
- 교사와 운영자가 읽기 쉬운 형태로 결과를 요약한다.

## 2. AI가 맡지 않는 역할

- 최종 점수 부여
- 최종 합격/불합격 결정
- 학생 감시
- 저작권/표절 최종 판정

## 3. AI 활용이 필수인 이유

### 개인별 질문 생성

- 고정 질문 세트로는 얕은 답변을 쉽게 우회할 수 있다.
- 학생 제출물마다 다른 질문을 생성하려면 LLM 기반 추론이 유리하다.

### 의미 비교

- 제출물과 답변은 같은 단어를 쓰더라도 이해 수준이 다를 수 있다.
- 키워드 매칭보다 의미 비교가 중요하다.

### 오개념 구조화

- 학생 개인 결과만으로는 교사에게 충분하지 않다.
- 반복되는 misunderstanding pattern을 묶어야 수업 개선에 쓸 수 있다.

### 사람 친화적 출력

- 점수만 내놓으면 위험하다.
- 교사가 읽는 판단 근거 문장과 운영자가 보는 집계 데이터가 함께 필요하다.

## 4. 권장 모델 역할 분리

### 경량 모델

- 질문 생성 초안
- 답변 길이/형식 검증
- 간단한 전처리
- 의심 사례 1차 분류

### 고성능 모델

- 의미 비교가 중요한 정밀 분석
- 논리 충돌 판별
- 근거 요약 품질 향상이 필요한 사례
- 경량 모델 결과가 불확실한 사례

### 사람 검토

- 최종 판정
- 불확실 사례 해소
- 결과 메시지 최종 통제

## 5. 프롬프트 설계 원칙

- 프롬프트는 목적별로 나눈다.
- 질문 생성 프롬프트와 분석 프롬프트를 분리한다.
- 프롬프트는 자연어 장문 한 덩어리보다 섹션형 구조가 좋다.
- 출력 형식은 명확한 JSON schema로 고정한다.
- 시스템 지시와 도메인 규칙을 프롬프트 내부에 섞어 쓰지 않고 레이어를 나눈다.

## 6. 프롬프트 권장 구조

### 질문 생성 프롬프트

- 제품 목적 요약
- 과제 설명
- 루브릭 핵심 개념
- 제출물 요약 또는 원문
- 질문 생성 규칙
- 금지 규칙
- 출력 스키마

### 분석 프롬프트

- 제품 목적 요약
- 분류 체계 설명
- 분석 기준 설명
- 루브릭 핵심 개념
- 제출물
- 질문
- 학생 답변
- 결과 서식
- 보수적 판정 규칙

## 7. 출력 스키마 강제

필수 원칙:

- 모든 AI 응답은 파싱 가능한 JSON으로 받는다.
- 파싱 실패 시 즉시 실패 처리하고 재시도한다.
- UI는 raw LLM 텍스트가 아니라 검증된 구조체만 소비한다.
- schema validation 통과 전에는 결과 저장 금지.

## 8. 재현성 전략

- 프롬프트 파일 버전 고정
- 모델 버전 기록
- 샘플 데이터셋 고정
- 결과 분류 taxonomy 고정
- 환경 변수와 런타임 설정 기록
- 분석 실행 시점과 request id 기록

## 9. 토큰 및 비용 전략

- 제출물 전문을 항상 그대로 넣지 않는다.
- 필요하면 전처리 단계에서 핵심 구간을 추린다.
- 질문 생성은 경량 모델을 우선 사용한다.
- 불확실하거나 고위험 사례만 고성능 모델로 escalated 한다.
- 집계와 정리는 가능한 한 규칙 기반 후처리로 보완한다.
- 동일 입력에 대한 중복 분석은 캐시한다.

## 10. 유지보수 전략

- 프롬프트를 코드와 분리한다.
- 도메인 분류 체계를 중앙 정의한다.
- 모델 호출 로직을 한 계층에 모은다.
- 출력 포맷을 스키마로 문서화한다.
- 샘플 입력/출력을 회귀 테스트에 사용한다.

## 11. 신뢰성 전략

- 불확실한 결과를 억지로 결정하지 않는다.
- 교사 최종 승인을 필수로 둔다.
- 판단 근거를 요약 문장과 구조화 필드 둘 다로 남긴다.
- 누락 개념과 충돌 문장을 별도 필드로 분리한다.

## 12. 리포트에서 강조할 표현

AI 리포트에는 아래를 강조한다.

- AI는 여기서 단순 챗봇이 아니라 `질문 생성기 + 의미 판독기 + 리포트 구조화 엔진` 역할을 한다.
- 효율성은 `모든 사례에 고성능 모델을 쓰지 않는 계층형 분석 전략`으로 설명한다.
- 재현성은 `프롬프트 버전`, `모델 버전`, `스키마`, `샘플 데이터`, `로그`로 설명한다.

## 14. 도구 기록 원칙

AI 리포트에는 실제로 사용한 도구만 적는다.

- 어떤 도구를 썼는지
- 어떤 모델을 썼는지
- 각 도구를 어떤 역할에 썼는지
- 어떤 제약을 두었는지
- 산출물 검증은 어떻게 했는지

과장하거나 사후에 맞춘 설명을 쓰지 않는다.
